# ADR-004: 배포 전략 결정

- **상태**: 대체됨 (→ ADR-005)
- **일자**: 2026-03-24
- **결정자**: 김진범

## 배경

시스템의 배포 및 운영 방식을 결정해야 한다. 안정성, 비용, 관리 용이성을 고려.

## 검토 선택지

### 1. Docker Compose (단일 VPS)
- 장점: 단순, 저비용 ($5~15/월), 관리 용이
- 단점: 단일 장애점 (VPS 다운 시 전체 중단)

### 2. Kubernetes (클라우드)
- 장점: 자동 복구, 스케일링, 모니터링
- 단점: 과잉 설계, 비용 높음 ($50+/월), 운영 복잡

### 3. 로컬 머신
- 장점: 비용 $0
- 단점: 전원/네트워크 불안정, 24/7 운영 어려움

## 결정

**선택지 1: Docker Compose (단일 VPS)**를 채택한다.

## 근거

1. **비용 효율**: $5~15/월. 수익이 검증되기 전 인프라 비용을 최소화
2. **단순함**: `docker-compose up -d` 한 줄로 전체 시스템 기동
3. **안전장치**: `stoploss_on_exchange: true`로 VPS 다운 시에도 거래소에서 손절 처리
4. **충분한 안정성**: 개인 자산의 1~5%만 투입. VPS 장애 시 최대 손실이 제한적
5. **자동 재시작**: `restart: always`로 컨테이너 크래시 자동 복구

Kubernetes는 수익이 검증되고 규모를 키울 때 마이그레이션 대상.

## 영향

- VPS 위치: 아시아 (일본/싱가포르, 바이낸스 DC 근접)
- 최소 사양: 2 vCPU, 2GB RAM, 20GB SSD
- 모니터링: Freqtrade 내장 + OpenClaw 브리핑으로 처리 (별도 모니터링 도구 불필요)
- 백업: user_data/ 디렉토리를 정기적으로 git push
