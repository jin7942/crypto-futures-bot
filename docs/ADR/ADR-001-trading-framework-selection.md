# ADR-001: 트레이딩 프레임워크 선정

- **상태**: 확정
- **일자**: 2026-03-24
- **결정자**: 김진범

## 배경

암호화폐 선물 자동매매 시스템에 사용할 오픈소스 트레이딩 프레임워크를 선정해야 한다. 핵심 요구사항: Binance Futures 지원, 백테스트, Python 기반 (ML/LLM 통합), Docker 지원.

## 검토 선택지

### 1. Freqtrade (48K stars, GPL-3.0)
- 장점: FreqAI 내장 (ML 최고), 가장 큰 커뮤니티, 문서 우수, 텔레그램/Web UI 내장, 선물 완전 지원
- 단점: GPL-3.0 라이선스, 전략 API 학습 곡선

### 2. NautilusTrader (21K stars, LGPL-3.0)
- 장점: Rust 코어로 백테스트 10~100배 빠름, 기관급 설계, 멀티에셋
- 단점: 학습 곡선 가파름, ML 모듈 없음, 크립토 지원 미성숙

### 3. Jesse (7.6K stars, MIT)
- 장점: 선물 전용 설계, 깔끔한 API, MIT 라이선스
- 단점: 라이브 트레이딩이 유료 플러그인, 작은 커뮤니티

### 4. CCXT + 직접 구축
- 장점: 최대 유연성, MIT 라이선스
- 단점: 백테스트/리스크 관리/주문 관리 모두 직접 구현 필요 (2~4개월)

## 결정

**Freqtrade**를 채택한다.

## 근거

1. **FreqAI**: ML/LLM 통합에 가장 적합. 다른 프레임워크는 ML 모듈이 없거나 미약함
2. **커뮤니티**: 48K stars, 활발한 업데이트 (당일 커밋 존재), 문제 발생 시 레퍼런스 풍부
3. **선물 지원**: Binance Futures isolated margin, 롱/숏, 레버리지 모두 지원
4. **Docker 공식 이미지**: `freqtradeorg/freqtrade:stable` 즉시 사용 가능
5. **텔레그램 내장**: 별도 알림 시스템 구축 불필요
6. **GPL-3.0**: 상업 배포 계획 없음 (개인 사용). 라이선스 제약 무관

NautilusTrader는 성능이 우수하나, 현 단계에서는 과잉. 규모 확대 시 마이그레이션 대상으로 보류.

## 영향

- Freqtrade 코어는 수정하지 않고 Docker 이미지로 사용
- 전략 파일(`user_data/strategies/`)만 커스터마이징
- upstream 업데이트는 `docker pull`로 처리
